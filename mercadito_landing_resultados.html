Diagnóstico del problema

Dolor / Oportunidad

  • Los líderes revisaban el Task Manager fila por fila para identificar quién cumplió el Fast Start — proceso lento y propenso a errores
  • Sin vista unificada de cumplimiento por turno, control y usuario
  • Cada líder interpretaba los datos de forma distinta — sin estándar
  • El incumplimiento pasaba desapercibido: solo 6% de cumplimiento — dato que antes era invisible

Nos propusimos

  • Dar visibilidad en tiempo real del cumplimiento del Fast Start por turno
  • Automatizar el análisis del export de Task Manager (Excel INVAS)
  • Generar un ranking accionable para que los líderes den feedback
  • Escalar la solución a toda la región LATAM
Qué construimos
📊
Mercadito Dash
Fast Start Control · MP · Implementado ARG
Dashboard operacional que analiza el export del Task Manager y muestra en tiempo real quién cumplió el Fast Start, quién no, y cómo evoluciona cada operador semana a semana.
📊
KPIs en tiempo real: Cumplimiento %, OK, Desvíos y Excluidos de un vistazo
🏆
Ranking automático: quién necesita feedback y quién no, ordenado por performance
📝
Resumen para compartir: texto listo para copiar al chat del equipo con un clic
📈
Comparativa semanal: evolución semana a semana por usuario para medir progreso
🔐
Zona Manager con PIN: configuración por site, targets, tolerancias, rutas excluidas
🌎
Soporte multipaís: MLA, MLC, COL, MLU — listo para escalar LATAM
🎬
Modo Demo con datos random para presentaciones y onboarding

Antes de Mercadito Dash este número no existía. La primera medición reveló que solo el 6% de los operadores cumplía el Fast Start — un problema operativo real que llevaba meses sin detectarse.

Visibilidad lograda. Por primera vez el equipo puede ver en tiempo real quién cumple el Fast Start y quién necesita intervención. El dato ahora existe.
⚠️
